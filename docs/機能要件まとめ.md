# Links システム 機能要件まとめ

> 対象者：後任担当者・開発引き継ぎ用  
> 更新日：2026年6月

---

## システム概要

**Links** は自動車販売店向けの統合業務管理システムです。  
顧客・車両の管理から、見積作成・受注処理・納車・売上計上までの一連の業務フローをカバーしています。

### 技術スタック

| 区分 | 技術 |
|---|---|
| フロントエンド | Next.js (App Router) + TypeScript + MUI v7 |
| バックエンド | Django + Django REST Framework |
| データベース | PostgreSQL |
| 認証 | JWT（Cookie ベース） |
| インフラ | Docker Compose + nginx + Let's Encrypt |

### 環境

| 環境 | URL | ポート |
|---|---|---|
| 本番 | https://links.hayasaka.co.jp | 80/443 |
| ステージング | https://staging.links.hayasaka.co.jp | 80/443（内部 8001/3001） |

## 主要機能一覧

### 1. 顧客管理

**できること**
- 顧客の新規登録・編集・削除
- 氏名・フリガナ・住所・電話番号・メールアドレス・顧客区分（個人/法人など）・担当者・地域・性別・生年月日の管理
- 顧客への画像・メモの登録
- 類似顧客の重複チェック（名前・電話番号）
- 過去の見積・受注履歴の確認
- 顧客データのCSVエクスポート
- Excelからの顧客一括インポート（`import_customers_from_excel` コマンド）
- 顧客に紐づく所有車両の管理（所有期間・現在の所有フラグ）

**画面パス**
- `/dashboard/customers` — 一覧
- `/dashboard/customers/new` — 新規作成
- `/dashboard/customers/[id]` — 詳細・編集

---

### 2. 車両管理

**できること**
- 車両情報の登録・編集（車名・メーカー・年式・型式・排気量・エンジン型式・車台番号・カラー）
- 車台番号の重複チェック
- 登録情報の管理（登録地域・ナンバープレート・型認番号・初年度登録・車検満了日）
- 任意保険情報の管理（保険会社・対人/対物/搭乗者/車両保険・オプション）
- 保証情報の管理（保証期間・プラン名）
- 車両への画像・メモの登録
- 顧客との所有関係の管理（誰がいつからいつまで所有していたか）

**画面パス**
- 顧客詳細画面内に車両情報を表示
- `/dashboard/customers/[id]/vehicles/[vehicleId]` — 車両詳細

---

### 3. 見積管理

**できること**
- 見積の新規作成・編集・削除
- 3つのタイプで作成可能
  - **車両購入**：車両情報・下取り車・登録情報・任意保険・納車予定をセットで管理
  - **整備・修理**：対象車両（顧客所有車両から選択 or 手入力）を指定
  - **車両なし（用品・作業のみ）**：商品・作業費のみの見積
- 明細の入力（その他費用・課税費用・非課税費用）
- 支払い方法の内訳入力（現金・カード・ローン・振込・QR・商品券・下取り）
- ローン詳細の入力（信販会社・分割回数・支払開始月・初回/2回目/ボーナス払い額）
- 見積書のPDF出力・印刷
- ステータス管理（下書き → 提出済み → 受注済み）
- 見積から受注への一括変換

**バリデーション（保存前にチェックされるもの）**
- 顧客氏名が必須
- 見積日が必須
- ローン金額がある場合、信販会社・分割回数が必須
- 支払い合計が請求額と一致しているか
- ローン設定は1件のみ

**画面パス**
- `/dashboard/estimates` — 一覧
- `/dashboard/estimates/new` — 新規作成
- `/dashboard/estimates/[id]` — 詳細（見積書表示）
- `/dashboard/estimates/[id]/edit` — 編集

---

### 4. 受注管理

**できること**
- 見積から受注を作成（顧客情報・明細を自動引き継ぎ）
- 受注の直接新規作成・編集
- 見積と同様の明細・支払い入力
- ステータス管理（下書き → 受注確定 → 納車 → 売上計上）
- 受注書のPDF出力・印刷
- 受注キャンセル申請（申請 → 承認/却下フロー）
- 管理者による強制削除

**画面パス**
- `/dashboard/orders` — 一覧
- `/dashboard/orders/new` — 新規作成
- `/dashboard/orders/[id]` — 詳細（受注書表示）
- `/dashboard/orders/[id]/edit` — 編集

---

### 5. 納車・入金管理

**できること**
- 受注ごとの納品状況の管理（未納品 / 一部納品 / 完了）
- 明細単位での納品記録
- 入金記録の登録・削除
- 月別の納車集計の確認
- 納車管理一覧のCSVエクスポート

**画面パス**
- `/dashboard/management` — 納車管理一覧
- `/dashboard/management/[orderId]` — 受注別 納車・入金詳細

---

### 6. スケジュール管理

**できること**
- 納車予定・業務予定のカレンダー表示
- スケジュールの新規作成・編集・削除
- 受注・見積・顧客との紐付け
- 担当スタッフ・店舗の割り当て
- 納車方法（店頭/配達など）・納車店舗の記録

**画面パス**
- `/dashboard/schedules` — カレンダー

---

### 7. 業務連絡

**できること**
- スタッフ間でのスレッド形式のメッセージ送受信
- 顧客と紐付けた業務連絡の管理
- スレッドのステータス管理（未対応 / 対応済み）
- 添付ファイルの送受信

**画面パス**
- `/dashboard/business-communications` — 業務連絡一覧

---

### 8. 売上分析

**できること**
- 日別売上の推移グラフ・集計
- 受注別の売上一覧
- 商品カテゴリ別の売上分析
- スタッフ別の実績分析

**画面パス**
- `/dashboard/analytics/sales` — 売上分析
- `/dashboard/analytics/staff` — スタッフ分析
- `/dashboard/analytics/product` — 商品分析

---

### 9. マスタ管理（設定）

**できること**
- 店舗の登録・編集・削除
- カテゴリの階層管理（最大5階層、車両/商品/費用/保険/その他）
- 商品マスタの管理（商品名・カテゴリ・メーカー・単価・税区分）
- 支払企業（信販会社・ローン会社など）の管理
- 会社情報（帳票に印刷される会社名・住所など）の設定

**画面パス**
- `/dashboard/settings/shops` — 店舗設定
- `/dashboard/settings/categories` — カテゴリ設定
- `/dashboard/settings/payment-companies` — 支払企業設定
- `/dashboard/settings/company` — 会社情報設定

---

### 10. スタッフ管理

**できること**
- スタッフアカウントの作成・編集・削除
- ロール（権限）の設定
- スタッフ一覧のCSVエクスポート・インポート

**画面パス**
- `/dashboard/staffs` — 一覧
- `/dashboard/staffs/new` — 新規作成
- `/dashboard/staffs/[id]` — 詳細・編集

---

### 11. キャンセル管理

**できること**
- 受注のキャンセル申請（理由を入力して申請）
- 管理者によるキャンセル承認・却下
- 承認されたキャンセルの取消
- キャンセル申請一覧の確認

**画面パス**
- `/dashboard/cancel-requests` — キャンセル申請一覧

---

### 12. 監査ログ

**できること**
- 全ユーザーの操作履歴の確認
- 操作対象・操作内容・変更前後の差分（JSON）の確認
- 実行者・日時・IPアドレス・ブラウザの記録

**画面パス**
- `/dashboard/audit-logs` — 監査ログ一覧

## 主要な業務フロー

```
顧客来店
  │
  ├─ 顧客登録（新規 or 既存検索）
  │
  ├─ 見積作成
  │     ├─ タイプ選択（車両購入 / 整備・修理 / 車両なし）
  │     ├─ 明細入力（車両・用品・諸費用）
  │     ├─ 支払い内訳入力（現金・ローン等）
  │     └─ 見積書PDF出力 → 顧客に提示
  │
  ├─ 受注確定（見積 → 受注に変換）
  │     └─ 受注書PDF出力
  │
  ├─ 納車・入金管理
  │     ├─ 納品記録（明細単位）
  │     └─ 入金記録
  │
  └─ 売上計上
        └─ 売上分析に反映
```

## エラーログの場所

バックエンドのエラーは以下のファイルに記録されます：

| ファイル | 内容 |
|---|---|
| `backend/logs/error.log` | ERRORレベル以上（500エラー・未処理例外） |
| `backend/logs/app.log` | WARNINGレベル以上 |

ローテーション：10MB × 最大5世代

確認コマンド（サーバー上）：
```bash
tail -f /opt/hayasys/backend/logs/error.log
```

## 開発・運用メモ

- ソースコードは `/opt/hayasys/` 以下に配置
- 本番・ステージング両環境を同一サーバーで運用（コンテナで分離）
- 本番へのデプロイ：`git pull` → `docker compose build` → `docker compose up -d`
- ステージングへのデプロイ：ブランチを切り替えて `docker compose -f docker-compose.staging.yml up -d --build`
- フロントの入り口（`/`）にBasic認証あり（URLはチームに確認）
